# Skill — GitHub

## Purpose

Version control, pull requests, and CI integration for code and documentation.

## When to use

- Documentation and DevOps publish changes via PR.
- Tester triggers or inspects CI workflows.

## Practices

- Branch names: `feature/<ticket>-short-slug`.
- PR template must include risk tier and correlation ID.
